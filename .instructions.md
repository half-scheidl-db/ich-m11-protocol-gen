# Project Description
This repository is a lightweight ICH M11 protocol draft generation toolkit.

ich_m11_renderer.py: converts Markdown into a styled HTML document and optionally renders it to PDF using WeasyPrint, with CSS and HTML template support.
assets: contains the ICH M11 HTML template, CSS styling, and JSON schema examples for protocol generation.
protocol_gen_ich_m11.ipynb: a demo notebook showing how structured clinical trial metadata can be used with an LLM to produce ICH M11-style protocol draft content, then transform that output into Markdown and renderable documents.
ich_m11_protocol_0.html and protocol_example.pdf: example generated outputs for the protocol rendering workflow.
In short, it’s a prototype for generating synthetic ICH M11 clinical trial protocol drafts from structured input and rendering them into HTML/PDF form.

# References

## Protocol template
 
File: references/ICH_Step4_M11_Final_Template_2025_1119.pdf 

This is the basis for the HTML template and styling used in the renderer, and serves as a reference for the expected output format of the generated protocols. It is a comprehensive document that outlines the structure and content requirements for clinical trial protocols under the ICH M11 guidelines.  The goal for the renderer is to produce outputs that closely match the formatting and organization of this reference document, ensuring that the generated protocols adhere to the standards set forth by ICH M11.
Due to LLM context limitations, the renderer focuses on replicating the formatting and structure of the reference document rather than generating content that fully meets all the detailed requirements. The example outputs demonstrate the renderer's ability to produce styled HTML and PDF documents that resemble the ICH M11 format, while acknowledging that the content may not yet be fully compliant with all guidelines.

When to use: read the template when developing the renderer to understand the formatting and structure requirements, and refer to it when evaluating the generated outputs to assess how closely they match the expected ICH M11 protocol format.
